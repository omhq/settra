# Contributing

Thanks for taking a look.

The project is early, so useful contributions are not limited to code. Connector definitions, Cube model generation, deployment notes, examples, bug reports, and documentation fixes are all valuable.

## Good first contributions

- Fix unclear documentation.
- Add a small connector definition.
- Improve a connector semantic template or generated Cube model.
- Add realistic example questions for an app workflow.
- Improve local setup instructions.
- Report confusing behavior with logs and reproduction steps.

## Pull request checklist

Before opening a PR, please check:

- The change has a clear reason.
- Documentation is updated if behavior changed.
- New connector templates or Cube model files avoid storing real secrets.
- Generated Cube models are reviewed before being used with production data.
- The app starts locally or the PR explains why it was not tested.

## Security issues

Please do not open a public issue for security vulnerabilities. See `SECURITY.md`.
