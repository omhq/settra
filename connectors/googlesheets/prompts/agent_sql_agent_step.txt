- For Google Sheets dynamic sheet tables, use live context table and column names exactly. Double-quote table or column identifiers that contain spaces, punctuation, uppercase letters, or start with digits, e.g. schema."Students" and "Student Name".
- For Google Sheets worksheet analytics, dynamic sheet tables are the user's normal row-level sheet data. Prefer them for record counts, lists, aggregations, and joins when present with real header columns or semantic metadata.header_row.
- Do not use googlesheets_cell to count customers/orders/records unless you reconstruct records by grouping cells into rows. COUNT(*) on googlesheets_cell counts populated cells.
- Use googlesheets_sheet for tab discovery and googlesheets_cell for raw cells, formulas, notes, hyperlinks, coordinate/range questions, or fallback reconstruction.
- Google Sheets dynamic tables use the configured/header row as column headers. If a dynamic table returns zero rows or has value-like column names, treat it as a bad header-row parse and query googlesheets_cell filtered by exact sheet_name. Reconstruct rows with row, col, and value instead of assuming the dynamic table is empty.
- Google Sheets sheet_name filters are exact text values. Use the exact title from googlesheets_sheet.title or verified context; do not change casing or guess title case.
- For raw-cell reconstruction, group cells by row and pivot known columns with expressions like max(value) filter (where col = 'A') as col_a. When no reliable header row exists, use neutral aliases such as col_a, col_b, col_c and describe the assumption.
